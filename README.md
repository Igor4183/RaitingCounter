# RaitingCounter

RaitingCounter — это C++‑проект для скачивания и парсинга страниц с результатами соревнований по ориентированию, генерацию HTML‑рейтинга.

Проект проходит несколько стадий обработки:

1. чтение `config.json`;
2. скачивание и парсинг страниц;
3. сбор и хранение структурированных данных;
4. генерация HTML‑файлов.

## Что умеет проект

- читает настройки из `config.json`;
- скачивает страницы с результатами соревнований;
- парсит HTML и извлекает нужные данные;
- формирует внутреннюю структуру данных по спортсменам и результатам;
- создаёт два HTML‑файла:
  - **debug** — простой вывод для проверки данных;
  - **release** — красивый HTML с UI для просмотра результатов.

## Технологии

- **C++**
- **Makefile**
- **nlohmann/json (****json.hpp****)**
- **HTML / CSS**
- терминальные команды macOS

## Требования

Для сборки проекта нужен:

- **macOS**
- **g++-14** (или другая версия компилятора, которую нужно указать в `Makefile`)
- `make`
- интернет-доступ для скачивания страниц

> Важно: проект использует терминальные команды macOS, поэтому вне macOS он может не собраться без доработок.

## Сборка

Сборка выполняется через `Makefile`:

```bash
make
```

Если в `Makefile` указан другой компилятор, нужно использовать именно его. Например, если там стоит `g++-14`, то компилятор должен быть установлен и доступен в терминале.

## Запуск

После сборки запустите бинарник:

```bash
./app
```

## Конфигурация

Проект читает настройки из файла `config.json`.

В конфиге хранятся:

- ссылки на страницы результатов;
- группы, которые нужно обрабатывать;
- тип итогового вывода;
- дополнительные параметры страниц (название, дисциплина, дата, класс и т.д.).

Пример логики конфигурации:

```json
{
  "valid_starts": 6,
  "groups": ["М18"],
  "pages": [
    {
      "id": 1,
      "url": "http://example.com/page.htm",
      "title": "Example Competition",
      "discipline": "classic",
      "date": "2026-04-05",
      "class": 1000
    }
  ]
}
```

## Выходные файлы

Проект может создавать два HTML‑файла:

- **debug HTML** — для быстрой проверки корректности парсинга;
- **release HTML** — для удобного просмотра результатов с оформлением.

## Структура проекта

Обычно проект состоит из нескольких модулей:

- `config` — чтение и обработка `config.json`;
- `parser` — скачивание и парсинг HTML;
- `model` — структуры данных;
- `generator` — генерация HTML‑страниц;
- `u32string` — работа с Unicode/кириллицей;
- `main` — точка входа.

## Примечания

- Проект рассчитан в первую очередь на локальный запуск.
- Часть логики завязана на терминальные команды macOS.
- HTML‑вывод можно использовать как для отладки, так и для красивого просмотра результатов.

Этот проект создавался как учебный и практический инструмент для:

- работы с файлами и потоками;
- парсинга HTML;
- работы с JSON;
- разбиения кода на модули;
- генерации HTML‑страниц;
